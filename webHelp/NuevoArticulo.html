Nuevo Artículo

 

Para crear un nuevo Artículo se debe presionar en el panel principal, el botón correspondiente a “NUEVO” señalado en un círculo azul, y se presentará la siguiente pantalla: 

 

 

Figura 1.23.  Nuevo Artículo 

 

En esta pantalla se debe registrar la siguiente información: 

  

Item:  

En este campo se debe colocar código de artículo a crear, y el sistema considerará los niveles determinados en la mascarilla, para su validación.  

 

Nombre (Corto):  

En este campo se debe colocar el nombre abreviado que identifique al producto. 

 

Nombre (Largo):  

En este campo se debe colocar el nombre largo que identifique al producto. 

 

Núm. Documento:  

En este campo se debe determinar si el artículo requiere almacenar número de documento, al momento de registrar una transacción. 

 

Fecha Vencimiento:  

En este campo se debe determinar si el artículo requiere almacenar fecha de vencimiento y en qué tipo de movimientos, al momento de registrar una transacción. 

 

 

1ª Letra Reservada:   

En este campo se debe colocar una letra reservada si se requiere así individualizar a este producto. 

 

Segundo Documento:  

En este campo se debe determinar si el artículo requiere almacenar segundo número de documento, al momento de registrar una transacción. 

 

Valor 2:  

En este campo se debe determinar si el artículo requiere almacenar un segundo valor, al momento de registrar una transacción. 

 

U. Medida Existencia:  

En este campo se debe colocar el código de la Unidad de Medida de Existencia, es decir con la cual se gestionará el inventario del producto internamente en la Compañía.  

 

U. Medida Compra:  

En este campo se debe colocar el código de la Unidad de Medida de Compra, es decir con la cual se registrarán los ingresos del producto por compras. 

 

Factor Conversión Compra:  

En este campo se debe colocar el factor de conversión entre las unidades de medida de existencia, en relación con la de compra, en el caso de ser la misma se deberá colocar el valor 1. 

 

U. Medida Venta:  

En este campo se debe colocar el código de la Unidad de Medida de Venta, es decir con la cual se registrarán los egresos del producto por ventas. 

 

Factor Conversión Venta:  

En este campo se debe colocar el factor de conversión entre las unidades de medida de existencia, en relación con la de venta, en el caso de ser la misma se deberá colocar el valor 1 

 

Días Aprovisionamiento:  

En este campo se debe colocar el número de días que se demora el aprovisionamiento (Tiempo del proceso de Compra) del producto. 

 

Clasificación ABC:  

En este campo se debe colocar el código de importancia que tiene el artículo, siendo la más alta “A”, y la más baja “C”. 

 

Origen del Artículo:  

En este campo se debe colocar el origen del producto, puede ser local, importado o servicio. 

          

Código Alternativo:  

Este campo sirve para colocar un código alternativo del producto que se requiera para el funcionamiento de la aplicación. 

 

Proveedor Primario:  

En este campo se debe colocar el código del proveedor primario, al cual se le compra el artículo. 

 

Código CPC:  

Este campo permite registrar el código del artículo con el cual será gestionado en el Plan de Compras. 

 

. 

Seguridad de Material:  

En este campo se debe colocar el nivel de seguridad que se debe proporcionar al artículo, esta puede ser Alta, Media o Baja. 

 

Gravable para IVA:  

Este campo sirve para determinar si el producto es gravable con IVA, al momento de comprar o vender. 

 

Manejar Lotes:  

En este campo se debe colocar si el inventario de el producto, necesita ser gestionado por lotes.
